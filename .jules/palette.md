## 2026-06-15 - Standardizing API Error Suggestions
**Learning:** In a headless backend context like an MCP server, UX translates primarily to Developer Experience (DX) and API consistency. When handling unknown actions, concatenating hints into a generic `error` string makes it difficult for client software to programmatically parse and surface alternatives. Extracting these into a dedicated `suggestion` key standardizes the API schema and drastically improves the DX for client implementers.
**Action:** When implementing error handling for user-facing API tools (like MCP actions), always place actionable recovery advice in a distinct, top-level JSON key (e.g., `suggestion`) rather than burying it within the `error` message string.
