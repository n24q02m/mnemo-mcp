## 2025-05-18 - Added context-aware `suggestion` to JSON error responses
**Learning:** In a backend/CLI-focused MCP server, traditional frontend UX paradigms translate to Developer/Agent Experience (DX). When the server returns raw errors without guidance, consumers (like LLMs or developers debugging tools) struggle to recover. Standardizing on a top-level `suggestion` key in JSON error responses makes the API significantly more actionable and "intuitive".
**Action:** When adding or refactoring error paths in server tool handlers (like `_handle_add`, `_handle_capture`, etc.), always ensure `_json({"error": ...})` includes a corresponding `"suggestion": "..."` field with concrete next steps.
## 2023-10-27 - Added robust fallback suggestions for NoneType inputs in MCP tools
**Learning:** When consumers (like LLMs or developers) provide no inputs (`action=None`, `topic=None`), tools often crash with unhandled exceptions (e.g. `TypeError: 'NoneType' object is not iterable` in `difflib.get_close_matches`). By guarding against `NoneType` and providing a dynamically generated fallback suggestion listing valid options, the API becomes much more resilient and explicitly guides the user back onto the "happy path."
**Action:** When adding fuzzy matching to error responses for invalid string inputs, always explicitly check for falsy/None values to prevent crashes and ensure a default "Available options are..." message is provided.
