## Phase 2: Intelligence Layer (v1.9.0)

### 2.1 Knowledge Graph (Lightweight, SQLite)

**New tables** (added via migration in `db.py`):

```sql
CREATE TABLE IF NOT EXISTS entities (
    id TEXT PRIMARY KEY NOT NULL,
    name TEXT NOT NULL,
    entity_type TEXT NOT NULL,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
CREATE UNIQUE INDEX IF NOT EXISTS idx_entities_name_type ON entities(name, entity_type);

CREATE TABLE IF NOT EXISTS relations (
    id TEXT PRIMARY KEY NOT NULL,
    source_id TEXT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
    target_id TEXT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
    relation_type TEXT NOT NULL,
    created_at TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_relations_source ON relations(source_id);
CREATE INDEX IF NOT EXISTS idx_relations_target ON relations(target_id);

CREATE TABLE IF NOT EXISTS memory_entities (
    memory_id TEXT NOT NULL REFERENCES memories(id) ON DELETE CASCADE,
    entity_id TEXT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
    PRIMARY KEY (memory_id, entity_id)
);
```

**New module**: `src/mnemo_mcp/graph.py`

Functions:
- `extract_entities(content: str) -> list[dict]` — LLM extracts entities + relations from text
- `upsert_entities(db, entities: list[dict]) -> list[str]` — insert/update entities, return IDs
- `create_relations(db, relations: list[dict]) -> None` — create entity relations
- `link_memory_entities(db, memory_id: str, entity_ids: list[str]) -> None` — link memory to entities
- `find_related_memories(db, memory_id: str, max_depth: int = 2) -> list[str]` — traverse graph
- `get_entity_context(db, entity_name: str) -> dict` — get entity with all relations + memories

**Entity extraction prompt**:
```
Extract entities and relations from this text. Return JSON:
{
  "entities": [{"name": "...", "type": "person|project|tool|concept|org|location|event"}],
  "relations": [{"source": "entity_name", "target": "entity_name", "type": "uses|works_on|related_to|depends_on|created_by|part_of"}]
}
```

**Integration points**:
- `memory add` → after insert, run entity extraction (async, non-blocking)
- `memory update` → re-extract entities, update links
- `memory delete` → cascade delete memory_entities (FK handles this)
- `memory search` → boost results that share entities with query matches (graph_boost)

**Graceful degradation**: Local mode (no LLM) → skip entity extraction entirely. Graph tables exist but empty. Search works as before.

### 2.2 Importance Scoring

**Schema change**: Add column to `memories` table:
```sql
ALTER TABLE memories ADD COLUMN importance REAL NOT NULL DEFAULT 0.5;
```

**Scoring logic** (in `_handle_add`):
- If LLM available: score 0.0-1.0 via prompt
- If local mode: default 0.5
- Prompt: "Rate importance of this memory for future recall (0.0=trivial, 0.5=normal, 1.0=critical). Return ONLY a number."

**Search scoring update** (final formula with all boosts):
```
With reranker + graph:
  score = rerank * 0.50 + recency * 0.15 + freq * 0.10 + importance * 0.15 + graph * 0.10

Without reranker:
  score = rrf * 0.50 + recency * 0.15 + freq * 0.10 + importance * 0.15 + graph * 0.10

Without graph:
  score = rerank/rrf * 0.55 + recency * 0.20 + freq * 0.10 + importance * 0.15
```

### 2.3 Auto-Archive

**New table**:
```sql
CREATE TABLE IF NOT EXISTS archived_memories (
    -- same schema as memories
    id TEXT PRIMARY KEY NOT NULL,
    content TEXT NOT NULL,
    category TEXT NOT NULL DEFAULT 'general',
    tags TEXT NOT NULL DEFAULT '[]',
    source TEXT,
    importance REAL NOT NULL DEFAULT 0.5,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    access_count INTEGER NOT NULL DEFAULT 0,
    last_accessed TEXT NOT NULL,
    archived_at TEXT NOT NULL
);
```

**Archive logic** (background task, runs daily or on sync):
1. Query: `SELECT * FROM memories WHERE last_accessed < datetime('now', '-{ARCHIVE_AFTER_DAYS} days') AND importance < {ARCHIVE_IMPORTANCE_THRESHOLD}`
2. Insert into `archived_memories` with `archived_at = now`
3. Delete from `memories` (CASCADE handles FTS, vector, memory_entities)
4. Log archived count

**Config**:
- `ARCHIVE_AFTER_DAYS: int = 90`
- `ARCHIVE_IMPORTANCE_THRESHOLD: float = 0.3`
- `ARCHIVE_ENABLED: bool = True`

**New action**: `memory(action="restore", memory_id="...")` — move from archive back to memories.

**New action**: `memory(action="archived", limit=20)` — list archived memories.

### 2.4 STM to LTM Consolidation

**Trigger conditions** (checked during archive cycle):
- Category has >10 memories with similar content (vector similarity > 0.7)
- Or manual: `memory(action="consolidate", category="...")`

**Pipeline**:
1. Group memories by category
2. For each category, cluster similar memories (vector similarity)
3. For clusters with >3 memories:
   a. LLM summarize cluster into 1 consolidated memory
   b. Set consolidated memory importance = max(cluster importances)
   c. Archive original memories (not delete — keep in archived_memories)
   d. Link consolidated memory to same entities as originals

**Consolidated memory format**:
```json
{
  "content": "[Consolidated from N memories] LLM summary...",
  "category": "same as originals",
  "tags": "merged unique tags",
  "source": "consolidation",
  "importance": "max of originals"
}
```

**Graceful degradation**: Local mode → skip consolidation (requires LLM for summarization).

### 2.5 Memory Dedup Detection

**On `add()`**, before inserting:
1. FTS search with new content → check top result similarity
2. If embedder available, vector search → check cosine similarity
3. Decision:
   - similarity > 0.9: Return `"Warning: Very similar memory exists (ID: xxx). Use update instead?"` + existing memory content
   - similarity 0.7-0.9: Return `"Note: Related memory found (ID: xxx). Proceeding with add."` + add normally
   - similarity < 0.7: Add normally (no warning)

**Config**: `DEDUP_THRESHOLD: float = 0.9`, `DEDUP_WARN_THRESHOLD: float = 0.7`

### 2.6 Configurable Temporal Decay

**Current**: `_RECENCY_HALF_LIFE = timedelta(days=7)` hardcoded in `db.py`.

**Change**: Move to config:
```python
recency_half_life_days: int = 7  # configurable via RECENCY_HALF_LIFE_DAYS
```

Pass to MemoryDB constructor, use in `_hybrid_score()`.
